PROBLEM 6.3: "Port 8000 already in use"
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  Cause: Another service using port 8000
  
  Solution Option 1: Stop other service
    Windows: netstat -ano | findstr :8000
    macOS/Linux: lsof -i :8000
    Kill process ID with highest number
  
  Solution Option 2: Change port in docker-compose.yml
    Find line: "8000:8000"
    Change to: "9000:8000"
    Then: docker-compose up -d
  
  Verify: curl http://localhost:8000 or http://localhost:9000

  Solution:
    1. Check space: df -h (Mac/Linux) or disk usage (Windows)

  If still failing:
    Delete other large files and try again
