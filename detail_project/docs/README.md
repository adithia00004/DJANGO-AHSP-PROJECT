# Folder `detail_project/docs`

Ringkasan fungsi setiap dokumen yang berada di folder ini.

| File | Fokus | Kapan Dibaca |
| --- | --- | --- |
| `FINAL_IMPLEMENTATION_PLAN.md` | Rencana akhir Jadwal Kegiatan (stack, fase, checklist deployment). | Saat menyusun roadmap atau hendak memulai fase baru. |
| `PHASE_1_IMPLEMENTATION_GUIDE.md` | Langkah operasional Phase 1 (setup Vite, modul baru, validasi). | Referensi ketika mengulang atau mengecek detail perubahan Phase 1. |
| `JADWAL_KEGIATAN_DOCUMENTATION.md` | Dokumentasi teknis halaman Jadwal Kegiatan (arsitektur, API, troubleshooting). | Untuk developer/QA yang mendalami fitur Jadwal Kegiatan. |
| `JADWAL_KEGIATAN_CLIENT_GUIDE.md` | Fokus client-side (Vite entry, modul JS, alur data, roadmap AG Grid). | Ketika mengerjakan kode front-end atau debugging modul baru. |
| `JADWAL_KEGIATAN_IMPROVEMENT_PRIORITIES.md` | Backlog prioritas + matriks impact/effort. | Saat menentukan fokus peningkatan berikutnya. |
| `FREE_OPENSOURCE_RECOMMENDATIONS.md` | Rekomendasi library open-source beserta alasan & lisensi. | Ketika mencari alternatif tool gratis. |
| `TECHNOLOGY_ALTERNATIVES_ANALYSIS.md` | Analisis perbandingan teknologi (Grid, Gantt, Export, dsb). | Untuk diskusi arsitektur atau evaluasi vendor. |

> Dokumen lain yang terkait (Quick Start, Roadmap, dsb) berada di akar repo dan dirangkum di `DOCUMENTATION_OVERVIEW.md`.
